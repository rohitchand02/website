Work

Coming soon...

Summer 2021, I interned on the T-Mobile For Business: Billing and Payments team. My team handled service history (payments, statements, data usage, etc.) for T-Mobile's B2B application. One of my main projects was to improve UI messages and handle errors caused when a brand new customer initially views their service history.

To handle the errors, the billing account number for the client needs to be looked up and checked for past service history. Once the output is received, the UI needs to receive this information and display the appropriate history in a graph or a message saying that the lookup had an error. The edge case we are interested in is when the lookup was successful but the client has no history due to not completing the first cycle of T-Mobile's service.

To make calls from the front-end to the back-end, where the billing information was stored, I had to implement 3 new API endpoints that retrieved the status code for the API. The status codes were then checked through a series of conditionals, which handled different cases of codes.

As mentioned above, the edge case we were most interested in was the one with status code 204 (No Content). Following this conditional, I updated the UI through 3 different paths to provide the client with the most accurate and useful information possible.

Due to this new set of error handling, I also had to slightly alter previous unit test values. These tests leveraged the Jasmine test framework, which was piped during CI/CD tests during commits/merges.

The most exciting part of this project was that towards the end of my internship, I was finally able to get my project into production! You can check out the presentation that I delivered at the end of my internship here.

Overall, I had a great experience with T-Mobile and working on my project. My team was very kind and patient with me throughout the entire process. I couldn't have asked for a better opportunity to kick-start my professional career.
